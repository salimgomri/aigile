# 🌍 Activités Universelles - Expérience Terrain

## Principe Coach Agile (30 ans terrain)

Certaines activités fonctionnent **quel que soit le problème d'équipe**. Ce sont des classiques éprouvés qui marchent toujours. Le système privilégie les **activités spécifiques au pattern détecté** (+30 points de score), mais garantit toujours une couverture complète avec les universelles.

- **Couverture** : Toutes les phases ont des universelles

## 🎯 Logique de Sélection

### 1. Pattern-Specific FIRST (Score +30)

```
Score de base : 50
Si activité spécifique au pattern : +30 = 80
Si activité universelle : 50
```

**Exemple** : Pour "silent-team"
- "Braver" (spécifique courage) → Score 80+
- "Dot Voting" (universel) → Score 50+

→ **"Braver" sera choisi en priorité**

### 2. Universelles en Backup

Si aucune activité spécifique n'est disponible pour une phase, les universelles garantissent une couverture complète.

**Exemple** : 'silent-team' n'avait aucune activité "Decide"
- Maintenant : 12 activités universelles "Decide" disponibles
- Dot Voting, SMART Goals, etc. sont toujours là

## 🎓 Règles Terrain Appliquées

### Quand privilégier les universelles ?

1. **Équipe débutante en rétro** : Mad Sad Glad, Dot Voting = valeurs sûres
2. **Contrainte de temps forte** : Timeline + One Word = rapides et efficaces
3. **Faible confiance** : ESVP anonyme + Feedback Door = sécurisants
4. **Grande équipe** : Dot Voting + Lean Coffee = scalent bien

### Quand insister sur les spécifiques ?

1. **Pattern fort détecté** : Braver pour silent-team, Speedboat pour burnout
2. **Problème ciblé** : 5 Whys pour repetitive-complaints
3. **Équipe mature** : Activités avancées spécifiques

## 🔧 Maintenance

### Ajouter une Activité Universelle

Éditer `lib/retro/activities.ts` :

```typescript
const UNIVERSAL_ACTIVITIES_IDS = [
  // ...existing...
  'retromat-XXX',  // Nouvelle activité universelle
]
```

**Critères** : L'activité doit fonctionner pour **tous les patterns** sans exception.

### Vérifier la Couverture

```bash
cd /Volumes/T9/aigile
node -e "
const { getActivitiesForProblem } = require('./lib/retro/activities.ts');
const patterns = ['silent-team', 'burnout', 'tensions'];
patterns.forEach(p => {
  const acts = getActivitiesForProblem(p);
  const byPhase = acts.reduce((acc, a) => {
    acc[a.phase] = (acc[a.phase] || 0) + 1;
    return acc;
  }, {});
  console.log(p, ':', byPhase);
});
"
```
